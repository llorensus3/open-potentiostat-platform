# Contributing

Thanks for contributing to Open Potentiostat Platform.

## Scope

Contributions are welcome in:
- analog hardware design
- firmware
- Raspberry Pi software
- calibration workflows
- documentation
- teaching materials
- validation datasets

## General principles

- keep changes modular
- document assumptions
- avoid untested claims about measurement accuracy
- clearly separate **prototype code** from **validated code**
- include comments where hardware-specific choices matter

## Pull requests

Please include:
- a short summary
- the motivation
- affected folders
- test notes or validation notes

## Issues

When opening an issue, please specify whether it concerns:
- hardware
- firmware
- software
- documentation
- teaching material
- validation / research workflow
